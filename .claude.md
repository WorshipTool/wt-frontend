# Claude Context – Frontend Only

This repository is FRONTEND ONLY.

## Stack

- Next.js
- TypeScript
- Jest for tests

---

## Core Rules

- This is UI / presentation code only
- No backend or database logic here
- All data comes from backend API

---

## API Usage

- API types are GENERATED from backend Swagger
- Generated files are READ-ONLY
- Never modify generated API types manually

---

## Testing Rules (IMPORTANT)

- NEVER run or suggest running E2E tests
- E2E tests are slow and not part of normal development
- Use ONLY:
  - Jest unit tests
  - Jest component tests (if applicable)

Assume:

- E2E tests are handled separately (CI or manual runs)
- Fast feedback is preferred over full coverage

---

## Working Style

- Focus on correctness and clarity
- Prefer simple solutions over complex abstractions
- Avoid unnecessary refactors

---

## Design System

- Landing page uses **Cyberpunk Neon** dark theme with animated neon effects
- Fonts: Orbitron (display) + JetBrains Mono (body), loaded via next/font
- Color palette: Neon cyan (#00e5ff), magenta (#ff00e5), purple (#7b2fff), green (#39ff14) on dark backgrounds (#0a0a0f)
- See `docs/Landing Page Design.md` for full design tokens, animations catalog, and component patterns
- Animations use Framer Motion + extensive CSS @keyframes (glitch, neon-pulse, scanner, grid-flow, etc.)
- GlitchText component provides chromatic aberration glitch effect for headings
- Background features multi-layered system: grid, perspective grid, scanlines, vignette
- Jest setup file at `jest.setup.ts` configures `@testing-library/jest-dom`
